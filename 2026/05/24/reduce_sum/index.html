一般的reduce算子的设计

  1. 把一个数组的数据, 分成若干块, 块的个数等于block的数目.
  2. 每一个block内部, 在自己的shared memory上进行reduce.
  3. 最后在block范围内进行reduce, 得到最终结果.

解决shared memory bank conflict

  1. Shared Memory的组织方式:
    • Shared Memory是一个block有一个.
    • Shared Memory分为32个banks, 每一个bank的大小是4字节.
  1. Shared Memory Bank Conflict的定义是:
    • 在同一个时钟周期, 对于同一个Warp内的所有线程, 他们访问内存地址 / 4, 然后%32 (映射到bank的编号), 如果有两个线程得到的这个数值相等, 那么就发生了bank conflict.
  2. 对于这种Reduce方式, 会存在bank conflict:
  • 0号线程访问smem[0]和smem[1].
  • …
  • 16号线程访问smem[32]和smem[33]. 16号线程的smem[32]和0号线程的smem[0]会发生bank conflict.
  1. 优化方式: 分组规约, 让0号线程访问smem[0]和smem[32].
    • 这种方式有一个约束, 就是block的大小必须是2的次幂.
